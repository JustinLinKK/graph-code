# Extension Gallery Example

This example gives GraphCode domain-specific material for extension-oriented demos. It includes embedded systems snippets and ML pipeline snippets that can be used with the built-in extension package settings.

## What to Scan

Open this directory in GraphCode:

```text
examples/extension-gallery
```

Suggested initialization:

- Project name: `Extension Gallery`
- Project description: `Small embedded and ML example for showing GraphCode extension-oriented block types.`
- Scanning instructions: `Group embedded firmware, telemetry bridge, device map, ML training pipeline, and experiment config. Highlight domain-specific inputs, outputs, buses, metrics, and artifacts.`

## Settings Demo

1. Open GraphCode settings.
2. Open the `Extensions` section.
3. Enable the embedded systems and ML pipeline packages if available.
4. Scan this example and show how the graph can represent domain concepts beyond generic files and functions.

## What to Screenshot

- Settings with extension packages visible.
- Embedded firmware nodes such as `sampleAdc` and `publishTelemetry`.
- ML pipeline nodes such as `train_model` and metric/artifact configuration.
